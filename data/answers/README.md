# answers/

Synthetic persona answers generated by local LLM, used for training and evaluating the field-prediction model.

## Files

| File | Description |
|------|-------------|
| `question_bank_answered_local.csv` | ~5,760 rows of synthetic answers. Each row is one (persona × question) pair annotated with `model_answer`, `model_score_1_to_5`, `predicted_fields`, `field_scores_json`, `confidence`, and metadata about the run. Produced by `scripts/answer_questions_local.py`. |
| `annotation_failures.jsonl` | Questions that failed annotation (timeouts, connection errors). Each line is a JSON object with `question_id`, `target_field`, `error`, `run_id`, and `timestamp_utc`. Used to identify gaps to retry. |

## Personas

Each question is answered by a synthetic expert persona (e.g. *Senior Frontend Engineer*, *Data Scientist*). The persona determines the expected field bias — this creates labeled training signal for the recommender.

## Regenerating answers

```bash
python data/scripts/answer_questions_local.py
# Requires a running Ollama instance (default: localhost:11434)
```

## Validating answers

```bash
python data/scripts/validate_synthetic_output.py
python data/scripts/quick_trust_check_local.py --sample-size 20
```

Outputs go to `reports/`.
