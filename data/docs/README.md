# docs/

Design documents, architecture notes, and implementation instructions. These are reference material — they describe decisions and designs, not runnable code.

## Files

| File | Description |
|------|-------------|
| `ADAPTIVE_QUESTIONNAIRE_INSTRUCTIONS.md` | Full implementation spec for the adaptive questionnaire system: phases, deliverables, data requirements, and evaluation criteria. Start here if building the recommender model. |
| `LABELING_SYSTEM_SUMMARY.md` | Summary of the multi-label classification system: label ontology design, coverage results, and methodology. |
| `LABEL_DISTRIBUTION_REPORT.md` | Analysis of label distribution across the question bank after automated labeling. |
| `NEURAL_MATCHER_DESIGN.md` | Design doc for the neural field-matching model that maps questionnaire answers to field probabilities. |
| `NEURAL_MATCHER_COUNCIL_VERDICT.md` | Review and approval notes for the neural matcher design. |
